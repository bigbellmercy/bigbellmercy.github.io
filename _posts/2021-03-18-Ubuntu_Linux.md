# Ubuntu Linux란?
Ubuntu(우분투)는 리눅스(Linux) 운영체제(OS)의 한 종류로서 현재 가장 널리 쓰이고 있다.
운영체제(OS, Operating System)는 컴퓨터에서 사용자와 프로그램이 실행되는 기반 환경을 제공하는 소프트웨어를 말한다.
리눅스 운영체제는 무료로 쓸 수 있고 소스 코드도 공개 되어 있으며, 로봇이나 각종 전자 장치들 속의 컴퓨터의 운영체제로 쓰이고 있다.
리눅스를 잘 쓸 수 있으면 공학 분야에서 기술적으로 성공하는 데에 도움을 줄 것이다.   
이 글은 우분투 18.04 버전을 기준으로 설명하므로, 다른 버전에서와 차이가 있을 수 있다.

# 프로그램 찾기

# 터미널
터미널(terminal)이란 사용자가 글로 명령을 내리는 창을 말하며, 윈도우 운영체제에서의 도스창 또는 명령창과 같은 역할을 한다.  
터미널을 쓰는 이유는, 숙달이 된다면, GUI 환경에서 각종 명령들을 내리는 것보다 빠르고 편하기 때문이다.
또한, 소프트웨어를 개발할 때에는, 여러 명령들과 프로그램들을 조합해서 실행해야 하기 때문에, 이는 GUI에서 하기는 어렵고, 이 터미널에서 스크립트 등을 활용하여 글(text)로 나타낸 명령을 실행하게 된다.  

## 프롬프트와 경로
터미널이 처음 실행되면, 보통, `사용자@컴퓨터:~$`로 나타나는데, 여기서, `:`과 `$` 사이에는 현재의 폴더(folder)(같은 말로, 디렉토리(directory)) 경로(path)가 표시된다.  
경로에서, `~`는 사용자 홈(home) 폴더를 뜻한다. 가장 상위 폴더가 루트(root) 폴더는 `/`로 나타낸다. `.`은 현재 폴더를 뜻하고, `..`은 상위 폴더를 뜻한다. 

## 파일 목록 보기
`ls`(list) 명령을 써서 폴더 속의 파일들의 목록을 볼 수 있다.  
`ls`를 쓰면 목록을 간략히 보여 주고, `ls -al`를 쓰면 목록을 자세히 보여 준다.

## 폴더 이동하기
`cd`(change directory) 명령을 쓴다.
현재 폴더 아래의 하위 폴더로 가려면 `cd folder`를 쓴다.
루트 폴더에서 부터의 절대 경로(path)를 쓰려면, `cd /some/folder`처럼 쓴다.
사용자 홈 폴더 아래의 폴더로 가려면, `cd ~/folder`를 쓴다.
경로를 잘 모르겠으면 상기 명령 중간에 tap 키를 두번 연속 빠르게 치면 가능한 폴더 이름들이 나타난다.

### 상위 폴더로 가기
`cd ..`

### 이전 폴더로 가기
`cd -`

### 사용자 홈 폴더로 가기
`cd ~`

## 복사하기
아래와 같이 `/source/file`라는 파일을 `/target` 폴더 속의 `file`이라는 이름으로 저장한다. 만약, 마지막 `file` 이름이 생략되면 같은 이름으로 복사된다.
`cp /source/file /target/file`

## 이동하기/이름바꾸기
파일을 이동시키는 명령과 이름을 바꾸는 명령은 같은 결과를 낳기에 둘다 `mv`(move) 명령을 아래처럼 쓴다. 
`mv /source/file /target/file`

## 파일 지우기
`rm`(remove) 명령으로 파일을 지운다.  
`rm file`  

## 폴더 지우기
다음 명령으로 폴더를 지운다. 여기서, `r` 옵션은 하위 폴더들도 지우라는 뜻이고, `f`는 쓰기 방지 된 것들도 지우라는 뜻이다.  
```
rm -rf folder`
```

## 파일 이름으로 찾기
다음 명령으로, <keyword>라는 단어가 이름에 들어간 파일을 찾는다.
```

```
 
## 파일/폴더 속성
파일과 폴더는 소유자(owner) 속성(property)과 읽기(read)/쓰기(write)/실행(execution) 속성을 갖는다.
`ls -al` 명령을 치면 아래처럼 파일들의 속성을 자세히 보여 준다. 이때, 첫 `d`라는 기호는 그것이 폴더라는 뜻이고, 파일이면 `-`로 표시된다.  
그 다음의 기호들은 `rwx`가 연속 세 번 반복되는데, `r`은 읽기 가능, `w`는 쓰기 가능, `x`는 실행 가능을 뜻하고, 해당이 없으면 그 위치에 `-`가 표시된다.  
`rwx` 세 벌들은 순서대로 `소유자 권한`, `그룹 권한`, `기타 권한`이다.  
`소유자 권한`은 이 파일/폴더를 소유한 소유자의 권한을 뜻한다.
`그룹 권한`은 그 소유자가 속한 그룹에 있는 다른 사용자들의 권한을 뜻한다.
`기타 권한`은 그 밖의 사용자들(손님(guest) 등)의 권한을 뜻한다.  
그 뒤에 처음으로 나오는 이름은 이 파일/폴더의 소유자이고, 그 다음 이름은 그룹이다.  
그 뒤의 숫자는 파일/폴더의 
```
user@computer:~$ ls -al
total 150059984
drwxr-xr-x 63 user jong         4096 Mar 30 16:06  .
drwxr-xr-x  3 root root         4096 Mar 11  2020  ..
drwxr-xr-x  9 user user         4096 Feb  2 14:59  Documents
-rw-r--r--  1 user user        48101 May 18  2020  2020-05-18-13-09-04.bag
-rwxrwxr--  1 user user      2373509 Mar 16  2018  flightplot.jar

```

# 파일 압축하고 풀기
## 탐색기를 사용해 압축하고 푸는 방법
파일 탐색기를 실행하고 압축하려는 폴더나 파일들을 선택한 뒤에 오른쪽 단추를 누르고 `Compress...`을 실행하여 압축한다.
압축을 풀려면, 압축된 파일을 선택하고 오른쪽 단추를 누르고, `Extract to`를 실행하여 푼다.  

## GUI 창에서, 암호화 압축하고 푸는 방법 (추천)
암호화 하기: 기본 설치된 Archive Manager를 찾아 실행하고, 압축하려는 파일을 끌어서 거기에 놓고, .zip 등을 고르고, 필요하면 Other Options에서 Password에 암호를 적고, Save를 눌러 암호화한다.  
암호 풀기: 풀려는 파일을 Archive Manager로 끌어 놓고, Extract 단추를 누르고, 필요시 암호를 입력하고, 폴더 위치를 골라서 푼다.  
윈도우에서 암호화된 압축 파일 풀기: 윈도우 파일 단축메뉴에서 풀거나 반디집 등으로 풀고 필요시 압호를 입력한다.  
 
## 터미널을 사용해 압축하고 푸는 방법
`tar`와 `xz`(또는 `gzip`) 명령으로 터미널에서 압축을 하거나 풀수 있다.

### 파일 하나를 압축하는 방법

> tar -cvf file.tar a.c

### 파일 여러 개를 압축하는 방법
> tar -cvf file.tar a.c b.c d.c

또는
> tar -cvf file.tar *.c

### 폴더를 압축하는 방법
압축 명령인 `xz`(또는 `gzip`)은 파일들을 압축할 수는 있지만, 폴더를 압축할 수는 없기에, 먼저 `tar` 명령을 써서 폴더들의 파일들을 하나로 묶어 주어야 한다. 따라서, 다음처럼 `foo`라는 폴더를 압축할 수 있다. 
> tar -cf - foo/ | xz -9 -c - > foo.tar.xz 

### 압축을 푸는데 파일명의 한글이 깨질 때 해결법
> unzip -O cp949 filename.zip

## SD 메모리 카드 복제하기
리눅스의 `dd` 명령을 쓰면, SD 메모리 카드에 설치된 모든 내용을 그대로 떠서 일반 컴퓨터에 이미지 파일로 저장해 둘 수도 있고, 이를 다른 SD 카드에 복제할 수도 있다.

### SD 메모리 카드의 이미지 뜨기
1. 먼저, 이미지를 뜨려는 SD 카드를 일반 리눅스 컴퓨터에 연결한다.
2. `sudo fdisk -l` 명령으로 이 SD 카드의 장치 이름을 알아 내야 한다. 이에는 `Disks` 소프트웨어도 도움이 된다. 이 `fdisk` 명령을 치면, `Disk /dev/sdb`처럼 나오는 것은 그 디스크 전체이고, 그 밑의 `/dev/sdb1`처럼 나오는 것은 그 속의 한 파티션이다. 디스크 전체를 이미지 뜨려면 `/dev/sdb`를 쓰고, 어느 파티션만을 이미지 뜨려면 `/dev/sdb1`을 쓴다. 여기서는, 디스크 전체의 이미지를 뜨는 것으로 가정하여 `/dev/sdb`를 쓰자.
4. 이 장치 이름을 가지고, `sudo umount /dev/sdb` 명령으로 장치를 언마운트 시킨다.
5. SD 카드의 이미지를 뜰 때, 빈 공간을 포함해서 SD 카드의 모든 용량을 그대로 뜨려면 아래 a)번을 하고(시간이 오래 걸림. 1GB당 10초 정도 소요됨), 실제 파일이 차지하고 있는 만큼만 이미지를 뜨려면 아래 b)번을 한다(이 경우, 부팅 가능한 SD는 복제후 부팅이 안됐음).

- a) 이 장치 이름을 가지고, 아래처럼 명령하는데, 여기서, `if`는 입력 파일, `of`는 출력 파일이고, `bs`는 한번에 읽고 쓰는 크기를 적는 옵션이고(생략하지 않으면 속도가 빨라진다), `status=progress`는 진행 상태를 표시하는 옵션으로서 생략될 수 있다. 시간이 지난 뒤에 `sd.img`라는 이름으로 컴퓨터의 홈 폴더에 SD 카드의 이미지 파일이 저장된다. 이제, SD 카드를 분리한다.
> sudo dd if=/dev/sdb of=~/sd.img bs=1M status=progress

- b) 파일 탐색기로 SD 카드의 디스크 아이콘의 팝업 메뉴에서 Properties를 실행하여, 이 디스크에 사용되는 용량을 알아 낸다. xxx GB used로 표시된다. (Contents: ... xxx GB가 아님에 유의한다). 그리고 다음의 명령으로 이미지를 뜬다. 여기서, bs=1M는 한번에 읽고 쓰는 용량의 단위이다. 이때, 아래의 `2048` 대신에 앞에서 알아낸 용량과 같거나 큰 크기를 입력한다.(아래 예에서는, bs가 1M로 설정되었기에 1M x 2048 = 2048 M = 2.048 GB가 된다)
> sudo dd if=/dev/sdb of=~/sd.img bs=1M count=2048 status=progress

### 이미지 파일을 새로운 SD 메모리 카드에 복제하기
1. 위에서 얻은 저장된 이미지를 새로운 SD 카드에 복제하려면, 새로운 SD 카드를 연결하고, `sudo fdisk -l` 명령으로 장치 이름을 알아 내는데, 장치 이름을 잘 못 알아내면 엉뚱한 하드 디스크 등에 덮어 써질 수 있으므로 무척 신중하게 확인해야 한다.
2. 예를 들어, 장치 이름이 `/dev/sdb`라면, `sudo umount /dev/sdb` 명령으로 장치를 언마운트 시킨다.
3. 아래 명령을 쓰면, 시간이 지난 뒤에, `sd.img` 이미지 파일이 `/dev/sdb`의 SD 카드에 복제된다. 여기서, `bs=1M`는 생략 가능하지만 적으면 속도가 빨라진다.
> sudo dd if=~/sd.img of=/dev/sdb bs=1M status=progress
4. 복제가 끝난 뒤에, SD 카드를 바로 빼지 말고, 시간이 2~3 분 지난 뒤에, 탐색기에서 이 드라이브에서 Unmount나 Eject를 누르고, 빼도 좋다는 문구가 나오면 그 때 뺀다. 왜냐하면, 이렇게 정상적으로 SD 카드를 꺼내지 않으면, 복제가 끝났어도, 젯슨 나노에 끼우면 부팅이 안되는 사례가 있었기 때문이다. 

## `apt` 명령으로 설치된 프로그램 찾기

> apt list --installed | grep program_name
